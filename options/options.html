Getting Started

Follow these steps to set up GitHub DevWatch

1

Create a GitHub Token

This allows the extension to fetch your repository activity securely.

â†’ Create a token with recommended permissions or manually create one with repo and read:user scopes.

Security: Your token is encrypted with AES-256 encryption and stored securely on your device. It's never transmitted to third parties.

Expiration: Tokens expire after 90 days by default. You'll need to create a new token when yours expires, or select "No expiration" when creating your token (not recommended for security).

2

Add Repositories

Go to the Repositories tab to add repositories you want to monitor. You can add them manually or import from your GitHub account.

3

Configure Filters

Visit the Activity Filters tab to choose which types of activity you want to see (Pull Requests, Issues, Releases) and enable notifications.

4

View Help & Changelog

Visit the Help tab to find documentation, support resources, and the changelog to see what's new in the latest version.

Tip: DevWatch checks for new activity every 15 minutes by default. You'll see a badge on the extension icon when there's new activity to review.

    Data Management

    Manage your extension data, cache, and settings

    Unlimited Repositories

    Allow watching more than 50 repositories

    Warning: We limit repositories to 50 by default to prevent exceeding GitHub's API rate limits and causing problems with your account. Enabling unlimited repositories may result in rate limit issues.

    Clear Cache

    Remove cached activity data. This won't affect your settings or repositories.

    Clear All Data

    Remove all cached data and activity history. Settings and repos are preserved.

    Reset to Defaults

    Reset all settings to default values. Your token and repositories will be cleared.

    Warning: Clearing data or resetting settings cannot be undone. Use the Export feature below to backup your configuration first.
